\section{Introduction}
\begin{frame}{Introduction}
\justifying
\begin{itemize}
    \item Traditional centralized cloud storage systems (like Google Drive, Dropbox) rely on central servers, creating a single point of failure and privacy risks.
    \item Decentralized Cloud Storage utilizes blockchain technology and IPFS (InterPlanetary File System) to secure files.
    \item Files are uploaded to IPFS, ensuring redundancy and decentralization, while the content hash (CID) is stored on the Ethereum blockchain.
    \item This system ensures data integrity, transparency, and gives users complete control over who accesses their files through smart contracts.
    \item It eliminates reliance on third-party intermediaries for data storage and management.
\end{itemize}
\end{frame}

% --------------------------------
% SLIDE 4 : SCOPE
% --------------------------------

\section{Scope}
\justifying
\begin{frame}{Scope of the Project}

\begin{itemize}
    \item Builds a Decentralized Application (DApp) that interfaces with the Ethereum blockchain and IPFS.
    \item Allows users to upload various file types (images, PDFs) which are stored on the Pinata IPFS network.
    \item Implements smart contracts using Solidity to manage file hashes and access control lists.
    \item Provides a secure sharing mechanism where users can grant or revoke access (`allow`/`disallow`) to specific wallet addresses.
    \item Ensures data immutability and retrievability without reliance on a central authority.
\end{itemize}

\end{frame}

\section{Relevance}
\begin{frame}{Relevance}
\justifying
\begin{itemize}

\begin{frame}{Proposed System - Overcoming Limitations}
\justifying
Our solution addresses the critical flaws of centralization by shifting control to the user and the blockchain.

\begin{itemize}
    \item \textbf{Eliminating Single Points of Failure:} Unlike central servers, we use \textbf{IPFS} for distributed, redundant storage that resists downtime and censorship.
    \item \textbf{Ensuring Data Ownership:} We replaced password-based logins with \textbf{Metamask}, granting users absolute ownership of their encryption keys and data.
    \item \textbf{Guaranteeing Transparency:} Start Contracts replace opaque database logs, providing an immutable, publicly auditable record of who shared what with whom.
    \item \textbf{Preventing Tampering:} Cryptographic hashing guarantees that any alteration to a file changes its ID, making undetectable data manipulation impossible.
\end{itemize}
\end{frame}

\section{Implementation Details}
\begin{frame}{Implementation Details}
\justifying
\begin{itemize}
    \item \textbf{Smart Contract (`Upload.sol`):}
    \begin{itemize}
        \item Developed in Solidity.
        \item Stores a mapping of user addresses to file URLs (`mapping(address=>string[])`).
        \item Manages access control via an `Access` struct and nested mappings.
    \end{itemize}

    \item \textbf{Frontend (React):}
    \begin{itemize}
        \item `FileUpload.js`: Handles file selection and uploading to Pinata via API.
        \item `Display.js`: Fetches and renders the user's images/files.
        \item `Modal.js`: Interface for sharing access with other users.
    \end{itemize}

    \item \textbf{IPFS Integration:}
    \begin{itemize}
        \item Uses Pinata API to pin files to IPFS and retrieve the unique CID.
    \end{itemize}
\end{itemize}

\end{frame}

\begin{frame}{Implementation Details - Workflow}
\justifying
\begin{itemize}
    \item \textbf{Workflow:}
    
    \vspace{0.2cm}
   
    \begin{enumerate}
        \item \textbf{Login:} User connects Metamask wallet.
        \item \textbf{Upload:} User selects a file, it is uploaded to IPFS.
        \item \textbf{Transaction:} The IPFS hash is sent to the `add` function in the smart contract via `ethers.js`.
        \item \textbf{Storage:} The contract stores the hash associated with the user's address.
        \item \textbf{Sharing:} User calls `allow` function to grant access to another address.
        \item \textbf{Retrieval:} `display` function checks permissions and returns the list of file hashes.
    \end{enumerate}

\end{itemize}

\end{frame}
